<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| child/player speech -> LiveKit AgentSession | Free-form user text simulates speech and can include unsafe requests, unrelated actions, or prompt-injection attempts. |
| LiveKit judge LLM -> test verdict | LLM judge evaluates behavior semantically, so prompts must be focused and explicit to reduce flaky or overbroad judgments. |
| expected-failure tests -> CI | Phase 2-dependent tests must preserve RED behavior without breaking Phase 1 CI usefulness. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Tampering | free-form player speech | high | mitigate | Add xfail behavior tests for prompt-injection and unsafe/out-of-adventure inputs that require refusal or redirect back to the predefined story. |
| T-01-02 | Information Disclosure | assistant response text | medium | mitigate | Add a behavior test that attempts to extract hidden/system/developer instructions and requires non-disclosure. |
| T-01-03 | Tampering | story progression behavior | medium | mitigate | Add tests requiring the assistant not to open the gate on incomplete attempts or after helper disclosure alone. |
| T-01-04 | Elevation of Privilege | kid-facing safety boundary | high | mitigate | Add safety redirect tests for unsafe content and require age-appropriate language for kids ages 10-14. |
| T-01-05 | Repudiation | test contract semantics | low | mitigate | Use explicit test names and judge intents tied to `V1_STORY` values so Phase 2 failures identify the broken behavior. |
| T-01-SC | Tampering | npm/pip/cargo installs | high | mitigate | No package installs or dependency changes are allowed; use existing LiveKit, pytest, pytest-asyncio, and ruff dependencies only. |
