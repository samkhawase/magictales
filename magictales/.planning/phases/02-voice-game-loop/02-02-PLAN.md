## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| player speech/text -> `evaluate_turn` | Untrusted user text controls story progression attempts. |
| `evaluate_turn` -> `GameState` | Classification decisions mutate explicit game state. |
| `Assistant` -> LLM | Runtime instructions and helper outputs can be attacked by prompt injection. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-05 | Tampering | `evaluate_turn` state mutation | high | mitigate | Safety and prompt-injection classification must run before objective matching; tests assert unsafe input leaves state unchanged. |
| T-02-06 | Elevation of Privilege | prompt-injection input | high | mitigate | Active LiveKit prompt-injection test must pass and response must not reveal internal/system/developer instructions. |
| T-02-07 | Information Disclosure | tests and prompts | high | mitigate | Test prompts must use synthetic unsafe/injection strings only and must not include real credentials or hidden prompt material. |
| T-02-08 | Tampering | unsafe-plus-correct answer precedence | high | mitigate | Unit test `test_unsafe_content_blocks_completion_even_with_correct_answer` blocks completion when unsafe text and correct answer are mixed. |
| T-02-09 | Repudiation | evaluator outcomes | medium | mitigate | `TurnResult.kind` makes success, partial, unrelated, unsafe, and restart-like outcomes inspectable in deterministic tests. |
